# Mimir v0.4.0 — Case Management (evidence, export, search, timeline filters)

**Date:** 2026-06-18
**Status:** Approved design, ready for implementation plan

## 1. Scope & Framing

Four cohesive case-management features shipping as **v0.4.0** (new
backward-compatible features → minor bump). This completes the "Case Timeline &
Management" line from the roadmap; AI integration shifts to v0.5. All four build
on the existing append-only `Case`/timeline model.

1. Evidence file management (add, tag, hash, verify)
2. Export case to a shareable archive (tar.gz bundle, or JSON document)
3. Cross-case search + `cases` status filter
4. Within-case timeline filtering (`--type`, `--grep`)

Build/sequence order: evidence model → evidence commands → timeline filters →
search → export → docs.

## 2. Evidence Management (foundation)

New append-only `evidence.jsonl` per case, mirroring `timeline.jsonl`. The
authoritative state of each tracked file is **folded** from its records on load
(an `add` followed by later `tag` ops accumulates tags).

```go
type EvidenceRecord struct {
    Op     string   `json:"op"`               // "add" | "tag"
    Name   string   `json:"name"`             // filename within evidence/
    SHA256 string   `json:"sha256,omitempty"` // set on add
    Size   int64    `json:"size,omitempty"`   // set on add
    Source string   `json:"source,omitempty"` // original path on add
    Tags   []string `json:"tags,omitempty"`   // tags added by this record
    Time   string   `json:"time"`             // RFC3339
}
```

Storage mirrors the timeline decision:
- `evidence.jsonl` is append-only (never rewritten) — forensic integrity.
- Records fold into an in-memory `[]Evidence` view on case open. New `Case`
  methods: `AppendEvidence(rec EvidenceRecord) error`, `Evidence() []Evidence`,
  unexported `loadEvidence() error` called from `LoadCase`. Corrupt lines are
  skipped with a warning (same tolerance as `loadTimeline`).

Folded view type:
```go
type Evidence struct {
    Name    string
    SHA256  string
    Size    int64
    Source  string
    Tags    []string
    AddedAt string
}
```

Commands (`evidence`, alias `ev`) — all require an open case:
- `evidence add <path> [--tag a,b]` — if `<path>` is outside the case, copy it
  into `evidence/`; compute streaming SHA-256 (reuse the `builtins` hash logic);
  append an `add` record (with size, source, initial tags); **also append an
  `evidence_added` timeline event**. Rejects missing/non-regular sources;
  refuses to overwrite an existing same-named file whose hash differs (warns).
- `evidence tag <name> <tag>...` — append a `tag` record; append an
  `evidence_tagged` timeline event.
- `evidence` (no args) — list tracked evidence: name, short hash, size, tags.
- `evidence verify [<name>]` — re-hash the file(s) on disk and flag mismatches
  against the recorded hash (tamper/integrity check). Missing files are reported.

## 3. Export (`export`)

- `export [path]` → `<case>.tar.gz` (default): bundles the whole case directory
  (`case.json`, `timeline.jsonl`, `evidence.jsonl`, `evidence/`, `output/`,
  `notes/`) plus a generated `manifest.json` at the archive root containing a
  case summary and the evidence name→sha256 list (so a recipient can verify
  integrity). Written to a temp file then renamed (no half-written archives).
  Never includes paths outside the case dir.
- `export --json [path]` → `<case>.json`: a self-contained document with
  metadata + full timeline + evidence manifest (names, hashes, tags). No binary
  files.
- Default output path: the current working directory (`<case>.tar.gz` /
  `<case>.json`); overridable by the positional `path` argument.
- Requires an open case.

## 4. Cross-case Search + `cases` Filter

- `search <query>` — scans **all** cases (does not require an open case),
  case-insensitive substring over: case name, note contents, tools used,
  evidence tags + filenames, and metadata values. Prints each matching case with
  a one-line "matched in: …" hint naming the fields that matched. Reads
  `case.json` + `evidence.jsonl` per case (not full timelines, for speed).
- `cases --status open|closed` — structured filter on the existing `cases` list.

## 5. Within-case Timeline Filtering (`timeline` flags)

Extend the existing `timeline` command (keeps the current `-n <N>` tail):
- `timeline --type <t>` — keep only the given event type(s):
  `tool_run|note|evidence_added|evidence_tagged|case_opened|case_closed`,
  comma-separated for multiple.
- `timeline --grep <substr>` — within-case search: keep events whose rendered
  summary / payload contains the substring (case-insensitive).
- Composition order: filter (`--type`, then `--grep`), then tail (`-n`).

## 6. Data Flow & Integration

Evidence and the timeline stay decoupled but cross-reference. `evidence add`/`tag`
write to `evidence.jsonl` **and** emit timeline events, so the investigation
narrative (`timeline`) shows when evidence entered the case, while `evidence`
shows the authoritative tracked-file state. Export reads both `case.json` and
the two `.jsonl` files. Search reads `case.json` + `evidence.jsonl` only.

New timeline event types introduced: `evidence_added`
(payload: `name`, `sha256`, `tags`) and `evidence_tagged`
(payload: `name`, `tags`). The `timeline` renderer gains branches for both.

## 7. Error Handling

- Preconditions: evidence/export/timeline commands require an open case;
  `search` does not. Clear messages otherwise.
- `evidence add`: missing/non-regular source → error; same-named file with a
  different hash already present → refuse + warn.
- `evidence verify`: hash mismatch → reported as a failure line; missing file →
  reported; clean → "ok".
- `export`: temp-file-then-rename; refuses if the case dir is unreadable; never
  archives paths outside the case dir.
- `loadEvidence`: corrupt `evidence.jsonl` line → skip + warn, keep the rest.

## 8. Testing

- **cases:** evidence record fold (add + later tag → current tags); corrupt-line
  tolerance; `verify` mismatch detection; `loadEvidence` on missing file → empty.
  All pure / temp-dir.
- **export:** round-trip a temp case → tar.gz → extract → assert the expected
  files exist and `manifest.json` hashes match; `--json` document shape.
- **search:** fixture cases → a query matches the right cases across each field
  (name, note, tool, evidence tag/filename, metadata); `--status` filter.
- **timeline:** `--type`, `--grep`, and `-n` compose correctly on a fixture
  timeline (filter-then-tail order).
- Existing suite stays green.

## 10. Deferred / Not in This Spec

- Encryption / signing of exported archives.
- Importing an exported case back into another Mimir instance (export is
  one-way for now).
- Full-text indexing for search (linear scan is fine at expected case counts).
- Evidence chain-of-custody beyond add/tag/verify (e.g. per-access audit).
