# Mimir v0.4.0 — Case Management, Investigation & UX

**Date:** 2026-06-18
**Status:** Approved design, ready for implementation plan

## 1. Scope & Framing

A single large v0.4.0 release (all backward-compatible additions → minor bump)
spanning nine feature areas across two themes. Everything builds on the existing
append-only `Case`/timeline model and the readline REPL.

**Investigation / case data**
1. Evidence management (`evidence`/`ev`)
2. Export (`export`)
3. Cross-case search (`search`) + `cases --status` filter
4. Timeline filtering (`timeline --type/--grep`)
5. More native built-in tools (`hexdump`, `entropy`, `decode`)
6. IOC extraction & tracking (`ioc`)
7. Case import (`import`)

**UX / presentation**
8. Starship-style prompt
9. Bordered box-table output via a native render helper

Build/sequence order (also the plan's task order): render helper (`internal/ui`)
→ prompt → evidence model → evidence commands → timeline filters → search →
export → import → built-in tools → IOC → docs.

## 2. Shared Infrastructure

### 2.1 Render helper — `internal/ui`
New package backing all table/list output. Native, no new dependency (uses
`golang.org/x/term`, already an indirect dep, for TTY + width detection).

- `Table{Headers []string, Rows [][]string, Align []Align}` → bordered
  Unicode box table (`┌┬┐│├┼┤└┴┘`), right-aligning numeric columns.
- **Degradation:** when stdout is not a TTY (piped/redirected) OR terminal width
  < table width, render plain tab/space-aligned columns instead of borders, so
  output never wraps into garbage.
- Status-icon + color helpers (`● open` green / `○ closed` dim), honoring
  `NO_COLOR`.
- Used by `cases`, `tools`, `timeline`, `evidence`, `status`.

### 2.2 Starship-style prompt
The REPL loop prints a **context line** above the input line each iteration;
readline's `Prompt` is just the short input marker `❯ ` (colored). This keeps
readline's prompt-width math trivial (no multi-line prompt) while giving the
segmented look.

- Context line segments (omitted when empty): user · `mimir` · case name · case
  status · counts (`N tools · M ev`). Nerd-font icons + color by default.
- **Graceful fallback:** when `NO_COLOR` is set or `MIMIR_ASCII=1`, drop icons
  and color and use plain text segments; the input marker falls back to `|> `.
- No case open → context line shows only user · `mimir`.

## 3. Evidence Management (`evidence`, alias `ev`)

Append-only `evidence.jsonl` per case, mirroring `timeline.jsonl`. Current state
of each tracked file is **folded** from its records on load.

```go
type EvidenceRecord struct {
    Op     string   `json:"op"`               // "add" | "tag"
    Name   string   `json:"name"`             // filename within evidence/
    SHA256 string   `json:"sha256,omitempty"` // set on add
    Size   int64    `json:"size,omitempty"`   // set on add
    Source string   `json:"source,omitempty"` // original path on add
    Tags   []string `json:"tags,omitempty"`   // tags added by this record
    Time   string   `json:"time"`             // RFC3339
}
```
New `Case` methods: `AppendEvidence(rec) error`, `Evidence() []Evidence`,
unexported `loadEvidence()` called from `LoadCase`; corrupt lines skipped with a
warning. Folded view:
```go
type Evidence struct { Name, SHA256, Source, AddedAt string; Size int64; Tags []string }
```

Commands (require an open case):
- `evidence add <path> [--tag a,b]` — if `<path>` is **outside** the case, copy
  it into `evidence/` (original untouched); if it is **already inside**
  `evidence/`, register it in place. Compute streaming **SHA-256** (reuse the
  `builtins` hashing), append an `add` record, emit `evidence_added` timeline
  event. Reject missing/non-regular sources; refuse to overwrite an existing
  same-named file whose hash differs (warn).
- `evidence tag <name> <tag>...` — append a `tag` record, emit `evidence_tagged`.
- `evidence` (no args) — box table of tracked evidence: name, short hash, size,
  tags.
- `evidence verify [<name>]` — re-hash file(s) on disk, flag mismatches/missing.

## 4. Export (`export`)

- `export [path]` → `<case>.tar.gz` (default): bundles the whole case dir
  (`case.json`, `timeline.jsonl`, `evidence.jsonl`, `ioc.jsonl`, `evidence/`,
  `output/`, `notes/`) plus a generated `manifest.json` at the archive root
  (case summary + evidence name→sha256 list). Temp-file-then-rename; never
  archives paths outside the case dir.
- `export --no-output` — skip the `output/` dir (when tool outputs are large).
- `export --json [path]` → `<case>.json`: self-contained document (metadata +
  full timeline + evidence manifest + IOCs), no binary files.
- Requires an open case. Default output path: cwd.

## 5. Cross-case Search + `cases` Filter

- `search <query>` — scans **all** cases (no open case required),
  case-insensitive substring over: case name, note contents, tools used,
  evidence tags + filenames, IOC values, and metadata values. Prints each match
  with a "matched in: …" hint. Reads `case.json` + `evidence.jsonl` + `ioc.jsonl`
  (not full timelines, for speed).
- `cases --status open|closed` — structured filter on the `cases` list.

## 6. Timeline Filtering (`timeline` flags)

Extend the existing command (keeps `-n <N>`):
- `timeline --type <t>` — keep only the given event type(s), comma-separated:
  `tool_run|note|evidence_added|evidence_tagged|ioc_extracted|case_opened|case_closed`.
- `timeline --grep <substr>` — keep events whose rendered summary/payload
  contains the substring (case-insensitive).
- Order: filter (`--type` then `--grep`), then tail (`-n`).

## 7. Built-in Tools (native Go, via `run`)

Add to the `builtins` registry (always `StatusReady`, no Docker):
- `hexdump <file>` — canonical hex + ASCII dump (offset, 16 bytes, gutter), like
  `xxd`. Streamed.
- `entropy <file>` — Shannon entropy (bits/byte, 0–8) over the whole file; flags
  high entropy (≥ ~7.5) as likely packed/encrypted. Streamed byte histogram.
- `decode <input|file> [--base64|--hex|--url]` — decode the input (or file
  contents) in the named encoding; with no flag, auto-detect (try base64, hex,
  url in turn). Prints the decoded bytes (printable) / hexdump (binary).

## 8. IOC Extraction & Tracking (`ioc`)

Append-only `ioc.jsonl` per case; deduped folded view on load.
```go
type IOCRecord struct {
    Type  string `json:"type"`  // ipv4|ipv6|domain|url|email|md5|sha1|sha256
    Value string `json:"value"`
    Source string `json:"source"` // file path or output name it came from
    Time  string `json:"time"`
}
```
- `ioc <file>` — extract indicators from a file (typically under `evidence/`).
- `ioc --from-output <name>` — extract from a captured tool output under
  `output/`.
- Extracts IPv4, IPv6, domains, URLs, emails, and MD5/SHA1/SHA256 hashes via
  regexes; dedupes against already-tracked IOCs; appends new ones to `ioc.jsonl`;
  emits one `ioc_extracted` timeline event (payload: counts by type, source).
- `ioc` (no args) — box table of tracked IOCs (type, value, source).
- Requires an open case for tracking; prints results regardless.

## 9. Case Import (`import`)

- `import <archive.tar.gz> [--as <name>]` — the inverse of `export`: extract a
  bundle into the cases directory.
- **Conflict handling:** never overwrites. If the case name (from the bundle, or
  `--as`) already exists, import under a non-colliding name (append
  `-imported`, then `-imported-2`, …) and report the chosen name. `--as <name>`
  sets the target name explicitly (still auto-suffixed if that also exists).
- After extraction, verify each evidence file's SHA-256 against `manifest.json`
  and report any mismatch (does not abort — surfaces integrity warnings).
- Rejects archives containing paths that escape the target dir (zip-slip guard).

## 10. Data Flow & Integration

Per-case append-only logs are the source of truth: `timeline.jsonl`,
`evidence.jsonl`, `ioc.jsonl`. Evidence and IOC commands write their own log
**and** emit a timeline event, so `timeline` narrates the investigation while
`evidence`/`ioc` show authoritative state. Export bundles all three logs +
files; import restores them. Search reads the three logs (not rendered
timelines). The `internal/ui` helper renders every list command; the REPL loop
owns the context line and delegates the input marker to readline.

## 11. Error Handling

- Preconditions: evidence/export/import-target/timeline/ioc-tracking need an open
  case; `search` and `import` (creating a case) do not. Clear messages.
- `evidence add`: missing/non-regular source → error; same-name different-hash →
  refuse + warn. `verify`: mismatch/missing → reported.
- `export`/`import`: temp-then-rename; zip-slip guard; hash-verify warnings.
- `decode`: undecodable input under the chosen/auto encoding → clear error.
- Corrupt `evidence.jsonl`/`ioc.jsonl` lines skipped with a warning on load.
- `internal/ui`: never panics on zero rows / overlong cells; degrades on narrow
  width.

## 12. Testing

- **internal/ui:** box rendering for known rows; right-alignment of numeric
  columns; plain fallback when not a TTY / width too small.
- **prompt:** context-line builder produces expected segments for
  open/closed/no-case and the ASCII fallback under `NO_COLOR`/`MIMIR_ASCII`
  (pure function; the readline wiring is smoke-tested).
- **cases:** evidence + IOC record fold; corrupt-line tolerance; `verify`
  mismatch; missing-file empty.
- **builtins:** `hexdump` known fixture; `entropy` ~0 for constant bytes, ~8 for
  random; `decode` round-trips base64/hex/url and auto-detects.
- **ioc:** extraction matches expected indicators per type from a fixture;
  dedupe; `--from-output` path.
- **export/import:** round-trip a temp case → tar.gz → import under a new name →
  files + manifest hashes match; conflict auto-rename; zip-slip rejected.
- **search:** fixture cases matched across each field; `--status` filter.
- **timeline:** `--type`/`--grep`/`-n` compose (filter-then-tail).
- Existing suite stays green.

## 14. Deferred / Not in This Spec

- Encryption/signing of exported archives.
- IOC enrichment / threat-feed lookups (offline extraction only here).
- Per-access chain-of-custody auditing beyond add/tag/verify.
- User-configurable prompt themes / segment selection (one built-in style +
  ASCII fallback for now).
- Report generation (Markdown/HTML) — candidate for a later release.
