# Mimir v0.3 — REPL Upgrade (history, line editing, autocompletion)

**Date:** 2026-06-18
**Status:** Approved design, ready for implementation plan

## 1. Goal & Scope

Replace the bare `bufio.Scanner` read loop with a `chzyer/readline`-backed input
layer. This fixes all four input bugs at once and delivers persistent history
plus full context-aware Tab completion. The `dispatch(line)` architecture and
every `cmd*` handler stay exactly as-is — only *how a line is read* changes.

**The four input bugs (all confirmed by live demo):**
1. Arrow keys inject raw escape codes (`^[[A`) into the line instead of
   navigating history / moving the cursor.
2. No line editing — can't move the cursor mid-line, no Ctrl+A/E/W/U.
3. Ctrl+C kills the whole program instead of cancelling the current line;
   Ctrl+D/EOF handling is abrupt.
4. Paste / multiline misbehaves, and un-interpreted escape sequences **cascade**:
   once garbage leaks into the buffer, later command output is corrupted
   (observed: `case: command not founde   notes   output`).

All four trace to one root cause: `bufio.Scanner` reads raw bytes with no line
discipline. readline owning the terminal in proper raw mode fixes the whole
class.

**Plus two adjacent fixes surfaced during design:**
- Stale startup banner hardcoded to `Mimir v0.1.0`.
- Unknown input (e.g. typo `lw`) shells out and rewraps the subprocess exit code
  as a noisy `error: exit status 127`.

**In scope:** readline integration; history wired to existing `config.HistoryPath`;
context-aware completer; correct signal/EOF handling; quiet passthrough errors;
banner fix.

**Out of scope (later versions):** Bubble Tea / rich TUI (v0.5), live fish-style
dropdown completion menus, syntax highlighting.

## 2. Architecture

```
App.Run()
  └─ readline.Instance (configured once)
        ├─ Prompt:        a.buildPrompt()  (refreshed each iteration via rl.SetPrompt — case context changes)
        ├─ HistoryFile:   cfg.HistoryPath  (load on start, append on submit)
        ├─ AutoComplete:  *Completer       (new: internal/shell/completer.go)
        └─ Interrupt/EOF handled in the loop
  loop:
    rl.SetPrompt(a.buildPrompt())
    line, err := rl.Readline()
    switch on err: readline.ErrInterrupt → continue; io.EOF → exit; nil → dispatch
```

- `Run()` in `internal/shell/shell.go` is rewritten around `readline.Instance`.
- `buildPrompt()`, `dispatch`, and all `cmd*` handlers are **untouched**.
- `App` gains one field: `rl *readline.Instance`.
- New file `internal/shell/completer.go` holds the completer and its sources.

**Library:** `github.com/chzyer/readline`, added as a **direct** dependency in
`go.mod`. (`golang.org/x/sys` is already an indirect dep, so the footprint is
small.) Chosen for built-in history-file persistence, Ctrl+R reverse search,
robust multiline/paste handling, and the PrefixCompleter model — and because a
readline (line-input) layer composes cleanly with Mimir shelling out to external
tools, whereas a full-screen TUI (Bubble Tea) or a persistent-render completer
(`go-prompt`) would fight streaming subprocess output.

## 3. The Completer (`completer.go`)

Built as a `chzyer/readline` **PrefixCompleter tree** — its native declarative
model. Static command names at the root; dynamic callbacks for live sources.
Each dynamic source is a small standalone function (so it is unit-testable
without a TTY); the completer holds an `*App` reference and closes over it.

| Context | Completes from |
|---|---|
| 1st word | command names: `help exit status case cases tools run install build use note clear timeline` |
| `run <x>` / `use <x>` | `Tools.List("")` names **+** built-ins (`hash` `strings` `file` via `builtins.List()`) |
| `install <x>` | `catalog.List()` entries **not** already installed |
| `build <x>` | installed Docker-backed tools (`Tools.List("")` filtered by `RunsInDocker()`) |
| `case` flags | `-n -o -c` |
| `case -o <x>` / `case -c <x>` | existing case names via `Cases.List()` |
| `run <tool> <arg>` / other path args | filesystem paths relative to cwd (cwd is the case dir when a case is open → naturally surfaces `evidence/`) |
| `timeline` | `-n` |

**File-path completion:** a small `filePathComplete(token string) []string` helper
lists directory entries of the token's directory part matching its base part;
directory entries get a trailing `/`. Relative to `os.Getwd()`.

**Tab UX:** press Tab → complete the common prefix; ambiguous → readline lists
candidates. No persistent dropdown.

## 4. History

`chzyer/readline` persists history to `HistoryFile` automatically. Config on the
`readline.Config`:
- **HistoryFile:** existing `cfg.HistoryPath` (`~/.mimir/.history`) — finally wired.
- **Dedup:** `DisableAutoSaveHistory` left off; consecutive duplicates skipped
  (readline default behavior). `HistorySearchFold: true` for case-insensitive
  Ctrl+R search.
- **Cap:** `HistoryLimit: 1000`.
- Blank lines are not dispatched and not recorded (preserves current behavior).
- Ctrl+R reverse incremental search comes free from readline.

## 5. Error / Signal Handling (the bug fixes)

The read loop switches on `Readline()`'s returned error:
- **`readline.ErrInterrupt` (Ctrl+C):** discard the current line, loop again with
  a fresh prompt — does **not** exit. (Fixes "Ctrl+C kills the program.")
- **`io.EOF` (Ctrl+D on an empty line):** clean exit, identical to the `exit`
  command (print "Exiting Mimir..." and return nil).
- **`nil`:** `strings.TrimSpace`; skip blanks; otherwise `dispatch(line)`, same
  error handling as today (including the existing `err.Error() == "exit"`
  sentinel from the `exit` command).
- Arrow keys, Home/End, Ctrl+A/E/W/U, and bracketed paste are handled internally
  by readline — fixes the escape-code garbage, no-line-editing, and paste/cascade
  bugs with no custom code.
- While an external tool / Docker run executes, it owns the terminal as today;
  readline governs only the prompt line, so streaming tool output is unaffected.

**Quiet passthrough errors:** `cmdShell` distinguishes a subprocess's nonzero
exit (already self-reported on stderr) from a launch failure:

```go
func (a *App) cmdShell(line string) error {
    cmd := exec.Command("sh", "-c", line)
    cmd.Stdout, cmd.Stderr, cmd.Stdin = os.Stdout, os.Stderr, os.Stdin
    if err := cmd.Run(); err != nil {
        var exitErr *exec.ExitError
        if errors.As(err, &exitErr) {
            return nil // subprocess already printed its own error to stderr
        }
        return err // failed to launch sh, etc.
    }
    return nil
}
```

This removes the noisy `error: exit status 127` for typos while still surfacing
genuine launch failures.

**Banner fix:** the `Run()` startup banner currently hardcodes `Mimir v0.1.0`.
Introduce a single `Version` constant (package `shell` or a small `internal`
location) and reference it from the banner so it stays correct. Value: `0.3.0`.

## 6. Testing

readline drives a real TTY, so the interactive layer is not unit-testable. The
valuable, pure logic is the completer and the passthrough fix:

- **`completer_test.go`** — table-driven tests on each dynamic source function:
  - command completion (prefix `""`, `"t"` → `tools timeline`, etc.)
  - `run` → tool names + built-ins
  - `install` → catalog entries minus already-installed
  - `case -o` → case names from a fixture manager
  - `filePathComplete` against a temp dir: matches by prefix, dirs get trailing `/`.
  Each source function takes its dependencies as parameters (tool list, case
  list, dir path) so no TTY or full `App` is needed.
- **`shell` test** — `cmdShell("false")` returns nil; `cmdShell("ls /definitely/nonexistent/xyz")`
  returns nil (shell self-reported); a genuinely unlaunchable case surfaces a
  non-`ExitError`.
- **Manual smoke checklist** (in the plan, not automated):
  ↑/↓ recalls history within a session and across restarts; Tab completes in each
  context (command, `run` tool, `install` catalog, `case -o` case, file path);
  Ctrl+C cancels the current line without exiting; Ctrl+D on an empty line exits;
  pasting a multi-word command does not corrupt the line or later output.

## 7. Docs / Roadmap

- Add `github.com/chzyer/readline` to `go.mod` as a direct dependency
  (`go get`, `go mod tidy`).
- `Version` constant set to `0.3.0`; banner references it.
- CHANGELOG `[Unreleased]`: readline-based REPL, persistent history, context-aware
  Tab completion, Ctrl+C/Ctrl+D fixes, quiet passthrough errors, banner fix.
- PLAN.md (local/gitignored): note v0.3 REPL upgrade delivered; autocompletion
  item ticked.

## 8. Deferred / Not in This Spec

- Live dropdown completion menus, syntax highlighting, multi-line command editing
  beyond paste — would require `go-prompt`/Bubble Tea and fight subprocess output.
- Completion of flag *values* beyond case names (e.g. `-n` numeric hints).
- Fuzzy "did you mean?" command suggestions (the rejected passthrough option).
